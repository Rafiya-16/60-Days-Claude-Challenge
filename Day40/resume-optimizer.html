System Prompt Design

The assistant is scoped narrowly to one job: reviewing and optimizing resumes for students/freshers against a target role. It's instructed to:

  • Always return strict JSON matching a fixed schema — no prose outside the object — so the UI can render deterministically without brittle text-parsing.
  • Score across four weighted dimensions (ATS compatibility, impact/quantification, clarity, keyword relevance) rather than one vague number, so feedback is actionable.
  • Separate strengths, gaps, and fixes into distinct arrays instead of one blob of feedback — this maps directly to distinct UI sections.
  • Never fabricate experience, metrics, or skills the candidate didn't provide — a hard constraint, since resume honesty matters more than a "better sounding" resume.
  • Handle edge cases explicitly: resumes under ~100 words, missing target role, or non-resume text pasted by mistake all get a defined fallback response instead of a hallucinated report.

UI Decisions

  • A circular score ring gives an instant verdict before the user reads anything — mirrors how ATS tools and credit-score apps work.
  • Four sub-score bars (ATS fit, impact, clarity, keywords) break the single score into causes, so users know exactly what to fix.
  • Strengths / gaps / fixes are visually distinct (color-coded glyphs) so a skimming user can triage in seconds.
  • The rewritten resume renders in a monospace "document" block with a one-click copy button, since that's the actual deliverable most freshers want.
  • Dark glassmorphic theme with emerald/gold/violet accents was chosen for a premium, non-generic "career tool" feel rather than a bland corporate form.

Edge Case Handling

  • Empty or too-short resume input → inline validation before the API call fires.
  • API/network failure → dedicated error state with retry, not a silent blank screen.
  • Model returns malformed JSON → caught and surfaced as a friendly error rather than crashing the render.

Future Extensions

  • Tools: connect a live job-postings API so the "target field" auto-suggests real keyword sets from actual listings instead of relying on the model's training data.
  • Memory: store prior analyses per user so returning users see score trends across resume versions ("+12 points since last week").
  • Multi-step workflow: chain this into a 3-step flow — analyze → rewrite → generate a matching cover letter — using the rewritten resume as context for the next call.
  • File upload: accept PDF/DOCX resumes directly with client-side text extraction instead of requiring paste.
